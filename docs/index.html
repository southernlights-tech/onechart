Overview

OneChart is a generic Helm chart for web applications. The idea is that most Kubernetes manifests look alike, only very few parts actually change.

Why OneChart? Because no-one can remember the Kubernetes YAML syntax, and most applications have similar deployment patterns.

Getting Started

Installation

Add the OneChart Helm repository:

helm repo add onechart https://chart.onechart.dev

Quick Start

Deploy a simple nginx application:

helm template my-release onechart/onechart \
  --set image.repository=nginx \
  --set image.tag=1.19.3

With Custom Configuration

Create a values.yaml file:

image:
  repository: my-app
  tag: fd803fc

vars:
  VAR_1: "value 1"
  VAR_2: "value 2"

ingress:
  annotations:
    kubernetes.io/ingress.class: nginx
  host: my-app.mycompany.com

Deploy with your configuration:

helm install my-release onechart/onechart -f values.yaml

Using OCI Repository

You can also use OneChart from an OCI repository:

helm install my-release oci://ghcr.io/gimlet-io/onechart --version 0.74.0 \
  --set image.repository=nginx \
  --set image.tag=1.19.3

Kong Plugins

OneChart includes built-in support for Kong plugins when using Kong as your ingress controller.

üîå Flexible Plugin System: Configure any Kong plugin with custom settings, from CORS and rate limiting to JWT authentication and custom transformations.

Quick Example

kong:
  plugins:
    cors:
      enabled: true
      plugin: response-transformer
      config:
        append:
          headers:
            - "access-control-allow-origin:https://myapp.com"
            - "access-control-allow-credentials:true"
    
    rate-limit:
      enabled: true
      plugin: rate-limiting
      config:
        minute: 100
        policy: local

Examples

Simple Web Application

image:
  repository: my-web-app
  tag: v1.0.0

vars:
  NODE_ENV: production
  PORT: "3000"

ingress:
  host: myapp.example.com
  ingressClassName: nginx
  tlsEnabled: true

resources:
  requests:
    cpu: 100m
    memory: 128Mi
  limits:
    cpu: 500m
    memory: 512Mi

API with Database and Kong Plugins

image:
  repository: my-api
  tag: v2.1.0

vars:
  DATABASE_URL: "postgres://user:pass@db:5432/mydb"
  REDIS_URL: "redis://redis:6379"

ingress:
  host: api.example.com
  ingressClassName: kong

kong:
  plugins:
    rate-limit:
      enabled: true
      plugin: rate-limiting
      config:
        minute: 1000
        hour: 10000
    
    jwt-auth:
      enabled: true
      plugin: jwt
      config:
        uri_param_names: ["token"]

probe:
  enabled: true
  path: "/health"

autoscaling:
  minReplicas: 3
  maxReplicas: 20

Static Site with CDN

image:
  repository: nginx
  tag: alpine

ingress:
  host: www.example.com
  annotations:
    nginx.ingress.kubernetes.io/rewrite-target: /
    nginx.ingress.kubernetes.io/ssl-redirect: "true"

resources:
  requests:
    cpu: 50m
    memory: 64Mi
  limits:
    cpu: 100m
    memory: 128Mi
